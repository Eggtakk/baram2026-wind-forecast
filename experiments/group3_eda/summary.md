담당: 탁예린 · 스크립트: `scripts/eda_group3.py`

## 1. 그룹3 정의 및 격자 선정

- KPX 그룹3 = UNISON 터빈 5기(wtg01~wtg05) 전체, 설비용량 21.0MW(21,000 kWh/h).
  (`scada_unison_train.csv`가 곧 그룹3 SCADA 데이터 — 별도 필터링 불필요)
- 터빈 centroid: (37.2752, 128.9714)
- 가장 가까운 LDAPS 격자: `12, 11, 7, 16` (평균 사용)
- 가장 가까운 GFS 격자: `5` (단일, 다음으로 가까운 격자와 거리 차이가 큼)

## 2. kpx_group_3 분포 / 결측 / 이상치

- 전체 26,304행 중 8,766행 결측 → **2022년 전체 + 2024-07-11, 2024-07-30 각 3시간(총 6시간)** 내부 gap.
- 유효 구간: 2023-01-01 ~ 2025-01-01 (17,538시간).
- 음수값: 0건.
- 설비용량(21,000kWh) 초과: 38건, 최대 21,130kWh(+0.6%) — 계측 오차 수준으로 판단, 클리핑 여부 검토 필요.
- 0값: 2,846건(16.2%) — 무풍 구간, 정상 범위로 보이나 장시간 연속 0 구간은 센서/커트인 문제 가능성 있어 추가 확인 권장.

## 3. 계절성 / 일중 패턴

- **월별 이용률(capacity factor)**: 12월 45.1%, 1월 40.5%, 11월 36.2% > 여름 8월 11.9%, 9월 9.7% — 겨울철 계절풍 영향으로 뚜렷한 연간 계절성.
- **시간대별 패턴**: 새벽~아침(0~5시) 평균 발전량이 가장 높고(6,300~6,700kWh), 정오~오후(11~16시)에 가장 낮음(4,400~4,600kWh) — 일중 변동도 뚜렷함(산악 지형 야간 활강풍 영향 추정).
- → 모델 feature에 month, hour(혹은 sin/cos 인코딩)를 반드시 포함할 것.

## 4. 예보-발전량 시차(lag) 관계

- 동시상관(lag=0) 기준: LDAPS ws10 r=0.772, GFS ws10 r=0.624, GFS ws100 r=0.625.
- **lag=0이 모든 변수에서 최고 상관** → forecast_kst_dtm이 실제 발전량과 이미 잘 정렬되어 있어 별도 시간 이동(shift)은 불필요. ±1h 근처도 상관이 크게 떨어지지 않아 근접 시간대 평균/롤링 feature는 유효할 수 있음.
- LDAPS(1.5km, 4격자 평균)가 GFS(0.25°, 1격자)보다 상관이 뚜렷하게 높음 → 그룹3은 LDAPS를 주 feature로, GFS는 보조/장기 추세용으로 활용 권장.
- **Lead-time(예보 시계) 별 상관**: 12h~35h 구간에서 상관이 0.71~0.84로 변동하며 21~24h 부근에서 최고치. 다만 이 데이터는 초기화가 1일 1회뿐이라 lead-time과 시간대(hour-of-day)가 1:1로 겹쳐 있어, 이 변동이 "예보 정확도 저하"인지 "시간대별 발전량 패턴 차이"인지 이 데이터만으로는 완전히 분리되지 않음 — 해석 시 유의.

## 5. 다음 단계 제안

2. SCADA(unison) 개별 터빈 파워 합산치와 kpx_group_3 라벨을 대조해 파이프라인 검증(sanity check).
3. 0값 연속 구간(정지/커트인) 별도 플래그 feature 검토.
4. 베이스라인 모델(LightGBM) 학습 후 위 feature들의 importance로 재검증.
